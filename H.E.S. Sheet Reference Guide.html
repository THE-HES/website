📊Stats — Single Source of Truth

Stats (years serving, families served, programs count) appear on the homepage stat bar and in blurbs across the site. They must only be defined once — in the Org tab. If they exist anywhere else, updates to one place won't update the other.

🏷️Status Values — What They Mean

The Status field controls how a program or event appears on the website. Here is exactly what each value does:

ACTIVE
The program is running right now and available year-round. No registration window — just show up or call.
Examples: Food Pantry, Fitness Center, Music School
OPEN
Registration is currently open. Seasonal programs where you can sign up right now.
Examples: Summer Camp (during reg season), After School Program
LIMITED
Registration is open but spots are filling fast. Creates urgency — shown with an orange badge.
Examples: Teen Unit, Camp Xtra (limited capacity)
UPCOMING
The program or event hasn't started yet. It's announced but registration isn't open or the date hasn't arrived.
Examples: Fall Festival (months away), a new program launching next season
INACTIVE
Manually hidden from the website. Use this to temporarily remove a program without deleting it from the sheet.
Examples: A program that's on pause, an old registration link
EXPIRED
Automatically set by the website when the event's date passes. You never need to set this manually — it happens on its own via the valid_until column.
Examples: Annual Street Fair 2026 (June 7 — now expired)
⚠ "Upcoming" with a past date is a contradiction If a program is marked "Upcoming" but its date has already passed, that's a data error. Either update the date, change the status to "Active," or let it auto-expire (for Events). The Annual Street Fair and Camp Opening Day were both marked Upcoming but their dates had passed — they've been removed from the live events list.
⏰Auto-Expiry — How Events Disappear Automatically

You never need to manually delete old events from the Google Sheet. The website checks the valid_until date on every page load and hides any event where that date has passed.

You add an event row to the Events tab
Set valid_until = the last day the event should be visible (usually the event date itself). Example: 2026-10-13 for the Fall Festival.
Event appears on the website automatically
The website shows it in the events section and community events page as long as today's date is before valid_until.
The valid_until date arrives
On the day of the event (or the day after — your choice), the website stops showing it. No action needed from you.
The row stays in the sheet — safely archived
Old event rows are not deleted. They stay in the sheet as a historical record. They simply don't appear on the website anymore. You can always update the valid_until date to bring an event back.

📋Tab Anatomy — What Each Tab Does
Org
fact_key | fact_value | notes
🏛️ Organization identity — name, address, phone, website
📊 Stats bar numbers — stat_1_num through stat_3_num
🏷️ Tagline, mission phrase, population served
⏰ Building hours — hours_open / hours_close
⚠ Stats go HERE and only here
HeroSlides
active | theme | eyebrow | body | image_url | … | expiry_date
🖼️ Homepage hero carousel — one row per slide
Flat format (not EAV) — each column = one attribute
active = yes/no to show or hide a slide
expiry_date = slide auto-hides after this date
Use | in Theme to control line breaks: Summer Camp | 2026 is | now open.
Programs
entity | fact_key | fact_value | type
🗂️ One entity = one top-level program (8 total)
EAV format — each attribute = its own row
Key facts: age_range_label audience schedule
Sub-programs listed: sub_program (one row each)
UI settings: ui_accent ui_order photo_url
SubPrograms
category | sub_program | fact_key | fact_value
📄 Rich content for individual sub-program pages
Key facts: summary description ages
schedule cost register
⚠ Category must match Programs entity exactly
"Social Services" ≠ "Social Service" — spelling matters
Events
event | fact_key | fact_value | valid_until
📅 Upcoming events — auto-expire via valid_until
EAV format — one event can have many rows
Key facts: date month start_time location
⚠ Always set valid_until — or it shows forever
Old events stay in sheet as archive — just hidden
Calendar
activity | start_time | end_time | room | days | notes
🕐 Recurring weekly activities — filtered by today's day
Flat format — one row per activity
days = Mon,Wed,Fri or Mon-Fri or just Tue
Only shows today's activities on the kiosk/website
⚠ Mostly empty — needs to be filled in
