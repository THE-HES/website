1 Task 1 — Content Audit Framework & Schema Design

Every text element rendered on any H.E.S. channel maps to exactly one row in the Master Sheet. No front-end should hard-code copy. The schema below is the contract between the Content team and Engineering.

2 Task 2 — Expiration Date & Content Disabling Logic

2A. Scheduling Metadata Columns

These six columns, added to the Master Sheet, give the content team full control over the lifecycle of any row without requiring an engineering deploy.

Column Type R/O Purpose Accepted Values / Format
publish_date ISO 8601 UTC O Earliest datetime channel may render this row 2026-06-29T09:00:00Z · Null = render immediately
expiry_date ISO 8601 UTC O Datetime after which channel must not render this row 2026-08-19T23:59:59Z · Null = never expires
timezone_ref IANA TZ String O The local timezone used when displaying date/time values in copy (e.g. "9:00 AM ET"). Does NOT affect expiry evaluation — that is always UTC. America/New_York
status Enum R Manual status override. Takes precedence over date logic. Use to emergency-disable content regardless of schedule. ACTIVE · DRAFT · SCHEDULED · EXPIRED · DISABLED
fallback_content_id String (FK) O Content row to render when this row is not renderable. Must be ACTIVE. Prevents blank component slots. Any valid content_id · Null = hide component
cache_ttl_seconds Integer O Hint to the API/CDN layer for how long this row may be cached. Set low (60–300s) for time-sensitive EVENT rows; higher for MICROCOPY. 60 · 300 · 86400 · Null = use channel default

2B. Render-Time Evaluation Logic (Pseudocode)

Every channel runs this decision tree when it fetches a content row. Evaluate in order — first failing condition wins.

// ─────────────────────────────────────────────────────────────
// FUNCTION: isRenderable(row, requestTimestampUTC)
// Called by: Web renderer, Kiosk app, Flyer tool
// Returns: { render: bool, copy: string | null }
// ─────────────────────────────────────────────────────────────

function isRenderable(row, now) {

  // STEP 1: Manual status override (highest priority)
  if (row.status === "DISABLED" || row.status === "DRAFT") {
    return resolveToFallback(row, now);
  }

  // STEP 2: Null / missing default_copy guard
  if (!row.default_copy || row.default_copy.trim() === "") {
    logWarning(`content_id ${row.content_id} has empty default_copy`);
    return resolveToFallback(row, now);
  }

  // STEP 3: publish_date — not yet live
  if (row.publish_date && now < toUTC(row.publish_date)) {
    return resolveToFallback(row, now);
  }

  // STEP 4: expiry_date — window has closed
  if (row.expiry_date && now >= toUTC(row.expiry_date)) {
    autoSetStatus(row, "EXPIRED");  // Write back to sheet via API
    return resolveToFallback(row, now);
  }

  // STEP 5: Passed all checks — content is live
  const copy = resolveChannelCopy(row);  // channel_override > default_copy
  return { render: true, copy };
}

// ─────────────────────────────────────────────────────────────
// HELPER: resolveToFallback
// ─────────────────────────────────────────────────────────────

function resolveToFallback(row, now) {
  if (!row.fallback_content_id) {
    return { render: false, copy: null }; // Component hides itself
  }

  const fallback = fetchRow(row.fallback_content_id);

  // Prevent infinite fallback chain (max depth: 2)
  if (!fallback || fallback.fallback_content_id === row.content_id) {
    logError(`Circular fallback on ${row.content_id}`);
    return { render: false, copy: null };
  }

  return isRenderable(fallback, now); // Recurse once
}

// ─────────────────────────────────────────────────────────────
// HELPER: resolveChannelCopy
// Priority: channel_override_[channel] > A/B variation > default_copy
// ─────────────────────────────────────────────────────────────

function resolveChannelCopy(row) {
  const override = row[`channel_override_${CURRENT_CHANNEL.toLowerCase()}`];
  if (override) return override;

  if (AB_TEST_ACTIVE && row.variation_a) return pickVariant(row);

  return row.default_copy;
}

1 Content Editor Edits default_copy, sets expiry_date, sets status = SCHEDULED Sheet UI validates char_limit before save.
2 Sheet → Webhook On-edit trigger fires webhook to API Google Sheets: Apps Script onEdit. Airtable: native Automation.
3 API / CMS Wrapper Invalidates CDN cache for affected content_id Use cache key = content_id + channel. Send purge to Cloudflare / Vercel edge.
4 Scheduled Job Every 5 min: scan rows where expiry_date <= now and status = ACTIVE; flip to EXPIRED Cron on API server. This is the backstop — channels should not rely solely on this.
5 Channel Renderer Calls isRenderable() at render time; never trusts cached status alone Always re-evaluate dates client/server side; cached status is stale.

2D. Edge Cases — Required Handling

HIGHTimezone Mismatch

Content team sets times in ET but sheet stores raw strings without timezone suffix. Channel reads as UTC, content expires 5 hours early (or late).

Fix: Enforce UTC-only in the sheet. Sheet UI auto-converts ET input to UTC on save via a helper column: expiry_date_et (display) → expiry_date (UTC, locked). Never store bare local time in expiry_date.
HIGHCache Serving Expired Content

CDN serves a cached page after expiry_date has passed. Event banner remains live hours after the event ended.

Fix: Set Cache-Control: s-maxage equal to cache_ttl_seconds. For EVENT rows, default TTL = 60s. Webhook on sheet edit triggers immediate CDN purge. Channel renderer always re-evaluates dates, does not trust cached status.
HIGHNull expiry_date = Permanent

Editor clears expiry_date intending to disable, but null means "never expires" — content stays live forever.

Fix: Null ≠ disabled. The status column is the only manual kill switch. Document this clearly in the sheet's frozen header row tooltip. Add a sheet validation rule: if content_group = EVENT and expiry_date is null → show warning cell.
MEDFallback Chain is Itself Expired

Row A expires → falls back to Row B → Row B is also expired. Component renders nothing with no warning.

Fix: resolveToFallback() recurses only once (depth = 2 max). If the fallback is not renderable, log an error and return { render: false }. The component must handle null copy gracefully — hide itself, never break layout.
MEDSheet API Unavailable

Google Sheets / Airtable API goes down. Channel fetches no data. All content slots are empty or broken.

Fix: API wrapper maintains a local Redis / edge KV cache of the last successful full sheet fetch. TTL = 15 min. Channel falls back to this stale cache. No sheet = stale content, not a blank page. Alert fires to ops on first 5xx from sheet API.
MEDCircular Fallback Reference

Row A's fallback_content_id = Row B; Row B's = Row A. Infinite loop crashes renderer.

Fix: resolveToFallback() checks if the fallback row's content_id === the original row's content_id before recursing. If circular, log error and return { render: false }. Add sheet validation to prevent self-referential fallbacks.
LOWpublish_date in the Past + status = SCHEDULED

Editor forgets to update status after scheduling. Content has a past publish_date but status is still SCHEDULED — it never renders.

Fix: Scheduled job that auto-flips SCHEDULED → ACTIVE when publish_date <= now (same cron as expiry check). Channel renderer also treats SCHEDULED rows as renderable if publish window is open, regardless of status — status is a manual override, not the primary clock.
LOWchar_limit Exceeded in Sheet

Editor saves copy that is longer than char_limit. Front-end truncates mid-word, breaking design layout.

Fix: Sheet-level validation (Apps Script / Airtable formula) blocks save and highlights the cell red if LEN(default_copy) > char_limit. API wrapper validates on ingest and rejects rows that violate char_limit with a 422 + message. Channel never truncates silently — logs a warning.

Quick Reference: Status Enum Decision Table

Status publish_date check? expiry_date check? Renders? Who sets it?
ACTIVE Yes — must be in window Yes — must not be past ✓ Yes Editor or auto-promotion from SCHEDULED
SCHEDULED Yes — must be in window Yes ✓ Yes (if window open) Editor pre-publishing
DRAFT No No ✗ Never Editor (WIP)
EXPIRED No No ✗ Never Cron auto-sets; editor may set manually
DISABLED No No ✗ Never (emergency kill) Editor only — manual override, bypasses all date logic
⚠ Engineering Implementation Note The status column is an override, not a cache. Channels must run the full isRenderable() check on every request — never shortcut to "status = ACTIVE → render." The date window must always be validated in real time. This is the only way to guarantee expired content is never served, regardless of sheet webhook latency or CDN cache state.
